@Github에 대하여 배운다
Git이란? 형상관리시스템 (Version Controll System)
게임의 세이브포인트와 유사하며 언제든지 저장시점으로 되돌아갈 수 있다
주로 개발자들이 프로그램과 관련된 파일을 저장하는데 사용

세가지 기본개념:
커밋(commit): 게임의 세이브에 해당하는 행동을 git에서는 커밋이라고 한다.
커밋을 하려면 저장을 원하는 파일을 묶어서 커밋 명령을 수행하면 된다.

스테이지 올린다(Add)
앞에서 커밋 하기 전에 저장을 원하는 파일을 묶는 일 이 작업을 스테이지에 파일을 올린다 혹은 줄여서 간단히 Add 라고 한다.

github에 업로드 (Push)
커밋을 하면 이제 현재 작업내용의 세이브 데이터가 내컴퓨터에 저장 되고
이걸 GitHub에 업로드 하면 마치 스팀이나 ps4 각종 모바일 게임의 원격 저장과 비슷한 역할을 한다.
다른 사람과 공유 할 수도 있고, 내 컴퓨터의 데이터가 날아가도 안전하게 복구할 수 있다.

예정:
1. github 가입
2. github 저장소 생성
3. github 저장소 클론
4. 내컴퓨터 내문서 아래에 생성된 프로젝트 디렉토리에서 파일 생성 및 작업완료
5. 커밋할 파일을 선택해서 스테이지에 올리기 :Add
6. Commit 하기
7. Push 하기
8. github.com/내아이디/내프로젝트 링크를 통해 정상적으로 업로드 되어 있는지 확인
